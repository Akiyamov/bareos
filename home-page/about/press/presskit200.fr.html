Texte du communiqué

4 Janvier 2007, Lausanne, Suisse: L'équipe de développement Bacula (Bacula Development Group) a annoncé aujourd'hui la sortie de la version 2.0 de la solution de sauvegarde réseau Bacula. Cette version apporte, comme demandé par les utilisateurs, des nouvelles fonctionnalités, de la maturité et des performances améliorées en fournissant une qualité et une mise en oeuvre comparables à une solution de sauvegarde orientée entreprise. La version 2.0.0 devrait entraîner encore davantage de migrations vers Bacula. La précédente version était la 1.38.11, publiée le 29 juin 2006.

"Bien que je n'ai pas fait d'étude détaillée, cette version est probablemment celle qui contient le plus grand nombre de nouvelles fonctionnalités" a déclaré Kern Sibbald, co-fondateur et membre principal de l'équipe de développement. "C'est en grande partie dü à l'énorme investissement de la communauté dans le projet. Merci à tous ceux qui ont rendu cela possible".

Nouvelles fonctionnalités visant à simplifier les sauvegardes réseaux :

  • Migration de Job (d'un volume vers un autre)
  • Premiére version de bweb, nouveau programme du package bacula-gui
  • Nouveau code de recherche sur disque, pour une restauration plus rapide des volumes sur disque
  • Chiffrement des données dans les Volumes
  • Support additionnel pour les périphériques extractibles, comme des DVD et disques USB.
  • La version Win32 utilise maintenant un cross-compiling via mingw32.
  • Le Director et le Storage Daemon ont maintenant été portés sous Windows.
  • Les programmes restore et bextract de bacula peuvent maintenant extraire les données non-portables Win32 vers n'import quel client (donc les Unix/Linux)
  • Il n'y a plus de limitation à 260 caractères pour les noms de fichiers Win32
  • Beaucoup d'améliorations pour lancer des scripts avant et aprés les tâches de sauvegardes, à la fois sur le client et le serveur.
  • Ajout du support des ACLs pour SunOS
  • Possibilité d'activer/désactier des volumes et des jobs
  • Support amélioré des DVD comme media de sauvegarde.
  • Amélioration du spooling quand plusieurs jobs l'utilisent en paralléle.
  • Les fenétres de statut de windows peuvent maintenant être redimmensionnées et parcourues.
  • Amélioration de la gestion des volumes pour l'utilisateur (localisation physique des bandes)
  • Nouvelles pages de man pour la plupart des démons et outils de Bacula

A propos de Bacula: Bacula est une suite de programmes pour gérer les backups, restaurations, et vérifications de données à travers le réseau. Ces programmes, chacun dédié à un jeu de tâches spécifiques, travaillent ensemble afin de de fournir une solution de sauvegarde robuste, facile à gérer et complète pour tout environnements. Bacula est le fruit du travail collectif de nombreux développeurs, depuis maintenant 6 ans. Bacula est un produit Open Source et les droits d'utilisations sont sont gratuits pour une utilisation commerciale ou non.

La qualité de Bacula a été prouvée par son inclusion dans les tutoriaux et livres traitant de backups. Son adoption est devenue trés vaste en une trés courte période de temps.

Interfaces graphiques

  • bimagemgr - Bimagemgr est une interface web écrite en Perl qui supervise les volumes disques devant être écrits sur CDROM
  • wx-console - wx-console est une console graphique écrite en wxWidgets et disponible sur toutes les plates formes clientes.
  • gnome-console - La gnome-console est une console graphique disponible sur les systèmes supportant Gnome 2.x.
  • tray-monitor - Le tray-monitor est un démon de surveillance qui réside dans le tray.
  • bweb - Bweb est une interface web qui fournit un outil pour les opérations basique et le reporting.
  • brestore - Brestore est un outil graphique de restauration, disponible sur les systèmes supportant Perl/GTK/Glade
  • bacula-web - Bacula-web est une interface web en PHP qui fournit un résumé des travaux déja exécutés.
